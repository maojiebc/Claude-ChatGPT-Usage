# Claude / ChatGPT 剩余额度色阶审查（v1.5.4）

## Audit scope

- 组件：Claude 与 ChatGPT 用量浮窗。
- 用户目标：无需读取细节，也能从颜色快速判断额度是否充足。
- 证据：中间档候选色板，以及本地预览 harness 中真实 userscript 的亮色/暗色截图。

## 1. 原问题：46% 仍显示绿色

- 截图：`01-current-green-at-46.png`
- 健康度：需要修复。
- 优点：百分比和进度条共用颜色，视觉映射一致；额度类型由图标区分。
- UX 风险：66%、71%、46% 全部为绿色，无法表达额度下降，46% 甚至会给出“充足”的错误暗示。
- 原因：旧函数先换算为已用量，并以 `used < 60` 为绿色，因此所有剩余量高于 40% 的额度都会显示绿色。

## 2. 中间档候选比较

- 截图：`07-middle-color-options.png`
- 健康度：已修正。
- 旧金黄 `#C37A04` 明显偏棕；鲜亮青柠 `#84CC16` 在白底上的对比不足；最终采用草木感更强的 `#4D9E0A`。
- `#4D9E0A` 与高额度翡翠绿 `#059669` 色相不同，在白底上的对比度约为 3.38:1。

## 3. Claude 亮色展开态

- 截图：`08-lime-expanded-v1.5.4.jpg`
- 健康度：良好。
- 86% 翡翠绿、66% 青柠绿、46% 橙色形成连续但不浑浊的健康度顺序。
- 百分比、进度条前景和柔和轨道同步变色；额度类型图标保持自身语义。
- 46% 橙色能表达“偏低但尚未告急”，强度比红色更合适。

## 4. 暗色 Claude 与 ChatGPT 同步

- 截图：`09-lime-dark-v1.5.4.jpg`、`10-chatgpt-lime-dark-v1.5.4.jpg`
- 健康度：良好。
- 暗色 Claude 中翡翠绿、青柠绿、橙色仍可区分；ChatGPT 的 66% 同步使用青柠绿。
- 控制台未发现 error 或 warning。

## 色阶规则

| 剩余额度 | 语义 | 颜色 |
| --- | --- | --- |
| 80–100% | 充足 | 翡翠绿 `#059669` |
| 60–79% | 关注 | 青柠绿 `#4D9E0A` |
| 40–59% | 偏低 | 橙色 `#EA580C` |
| 0–39% | 告急 | 红色 `#EF4444` |

## Evidence limits

- 本次使用本地 harness mock 用量接口，但浮窗 DOM、样式、状态机和渲染函数均为真实 userscript。
- 截图能确认可见对比和层级，不能单独证明完整 WCAG 合规；百分比文本提供了非颜色冗余信息。

## Recommendation

直接按剩余额度分档，并让 Claude 与 ChatGPT 共用同一函数。中间档避免棕黄，改用青柠绿；该建议已实施并通过自动化与视觉检查。
