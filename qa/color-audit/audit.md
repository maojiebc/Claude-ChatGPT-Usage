# Claude 剩余额度色阶审查

## Audit scope

- 组件：Claude 用量浮窗的收起态与展开态。
- 用户目标：无需读取细节，也能从颜色快速判断额度是否充足。
- 证据：用户提供的 46% 仍为绿色截图，以及本地预览 harness 中真实 userscript 的修复后截图。

## 1. 当前状态：46% 仍显示绿色

- 截图：`01-current-green-at-46.png`
- 健康度：需要修复。
- 优点：百分比和进度条共用颜色，视觉映射一致；额度类型由图标区分。
- UX 风险：66%、71%、46% 全部为绿色，无法表达额度下降，46% 甚至会给出“充足”的错误暗示。
- 原因：旧函数先换算为已用量，并以 `used < 60` 为绿色，因此所有剩余量高于 40% 的额度都会显示绿色。

## 2. 修复后收起态：86% / 66% / 46%

- 截图：`02-compact-palette.png`
- 健康度：良好。
- 86% 为绿色、66% 为金黄、46% 为橙色，三行在极小面积内仍能形成清晰的健康度顺序。
- 百分比文字保留，颜色不是唯一信息通道。

## 3. 修复后展开态

- 截图：`03-expanded-palette.png`
- 健康度：良好。
- 百分比、进度条前景和柔和轨道同步变色；图标颜色保持额度类型语义，不被健康度颜色覆盖。
- 46% 橙色能表达“偏低但尚未告急”，强度比红色更合适。

## 4. 低额度与深色主题

- 截图：`04-low-quota.png`、`05-dark-expanded-palette.png`
- 健康度：良好。
- 8% 与 15% 正确进入红色告警，55% 为橙色；深色主题下绿、金黄、橙仍可区分。
- 控制台未发现 error 或 warning。

## 色阶规则

| 剩余额度 | 语义 | 颜色 |
| --- | --- | --- |
| 80–100% | 充足 | 绿色 `#059669` |
| 60–79% | 关注 | 金黄 `#C37A04` |
| 40–59% | 偏低 | 橙色 `#EA580C` |
| 0–39% | 告急 | 红色 `#EF4444` |

## Evidence limits

- 本次使用本地 harness mock 用量接口，但浮窗 DOM、样式、状态机和渲染函数均为真实 userscript。
- 截图能确认可见对比和层级，不能单独证明完整 WCAG 合规；百分比文本提供了非颜色冗余信息。

## Recommendation

直接按剩余额度分档，并让 Claude 与 ChatGPT 共用同一函数。该建议已实施并通过自动化与视觉检查。
